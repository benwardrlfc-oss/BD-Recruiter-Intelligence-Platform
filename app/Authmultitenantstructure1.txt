Design the AI-powered parts of the platform so users cannot abuse token-heavy features or generate unnecessary cost.

I need strong usage control across all AI features such as:

Content Studio

BD Scripts

Candidate Matcher summaries

market briefings

signal explanations

e-shot generation

Implement the following principles:

1. Usage limits

Add hard daily and monthly limits by feature.
These limits should work at:

user level

organisation level

plan level

2. Rate limiting and cooldowns

Add backend-enforced cooldowns for expensive actions such as:

content generation

script generation

refresh actions

candidate parsing

3. Caching

Before calling AI, check whether a valid recent output already exists for the same request.
Reuse cached outputs wherever possible.

4. Structured generation flows

Do not allow unlimited open-ended generation.
Use controlled generation inputs such as:

tone

structure

company

signal

candidate profile

market scope

This keeps outputs high quality and reduces waste.

5. Usage tracking

Log AI usage for every request, including:

organisation

user

feature

model

tokens

estimated cost

timestamp

cache hit / miss

6. Budget enforcement

Support warnings and hard stops when usage exceeds allowed limits.

7. Precompute wherever possible

Daily briefings, watchlist summaries, and other repeated outputs should be generated in scheduled background jobs and stored, rather than regenerated on every refresh.

8. Abuse prevention

Add protection against excessive repeated generation and suspicious usage patterns.
Trial users and low-tier plans must have tighter limits.

The goal is to make the system feel intelligent and smooth while preventing users from burning large amounts of tokens unnecessarily.
